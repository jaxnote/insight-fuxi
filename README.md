**1.2、核心功能**
| 功能模块   | 功能描述                | 优先级 |
| ------ | ------------------- | --- |
| 自然语言分析 | 用户用自然语言提问，自动生成分析    | P0  |
| 分析路径规划 | 自动拆解分析问题，规划分析步骤     | P0  |
| 洞察解读   | 自动解读数据，生成业务洞察       | P0  |
| 分析能力库  | 预置通用分析方法（同比、环比、归因等） | P0  |
| 语义层配置  | 配置指标、维度、业务语义        | P1  |
| 跨场景迁移  |通过配置接入新业务领域          |P1    |
|多轮分析     |复杂问题多轮对话、自我修正      |P1    |
|可视化报告   |自动生成分析报告和图表          |P2    |



# 2、技术架构

平台构成：

- openclaw：负责编排
- agent：负责动态执行
- skills：负责提供标准的SOP，包括时序分析、归因拆解、对比分析几类，可能需要细拆分
- mcp：负责提供一些数据服务，包括专业术语RAG、指标检索RAG、元数据查询服务、非结构化知识检索服务（比如一些业务文档、业务SOP）

# 3、工作流程

Plan-Execute：识别需求类型，

- 如果是探索性需求：需要先沟通清楚需求和业务流程，再去选择合适的agent、skils、mcp完成任务，重在深度探索。
- 如果是确定性需求：确定需求没问题，直接执行即可。
